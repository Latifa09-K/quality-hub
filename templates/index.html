      What We Learned

      Test Cases

      A test case is a document that defines how to test a specific part of the system to ensure it works correctly. We learned about components like ID, steps, expected result, actual result, and status (Pass/Fail).

      • Positive & Negative Scenarios
      • Preconditions and Test Data
      • Execution and Validation
      ▶ Watch Test Case Tutorial

      Bug Reports

      We practiced how to report bugs clearly and reproducibly. A bug report includes title, steps to reproduce, expected vs actual result, environment, and evidence.

      • Bug Life Cycle: New → Assigned → Open → Fixed → Retest → Closed
      • Priority vs Severity classifications
      • Reproducibility ratio and attachments
      ▶ Watch Bug Report Guide

      Selenium

      Selenium is an open-source framework for browser automation. It supports multiple languages (Java, Python, C#) and integrates with Jenkins, Docker, and more.

      • Advantages: Free, Cross-Browser, Multi-Language, CI/CD friendly
      • Disadvantages: Web-only, requires coding, no built-in reports
      ▶ Watch Selenium Tutorial

      Playwright

      Playwright is a modern end-to-end testing tool that supports multiple browsers, mobile emulation, and parallel testing. It’s more reliable for modern web apps compared to Selenium.

      • Cross-browser support (Chromium, Firefox, WebKit)
      • Powerful selectors and auto-waiting
      • API testing integration
      ▶Watch Playwright Tutorial

      Postman

      Postman is a collaboration platform for API development. It allows creating collections, testing endpoints, and automating requests.

      • Design and test APIs with collections
      • Use environments and variables
      • Monitor APIs for uptime
      ▶ Watch Postman Official Tutorial ▶ Watch Postman Tutorial Playlist

      JMeter

      JMeter is a powerful open-source tool used for performance and load testing. It helps QA engineers measure system speed, stability, and scalability before production deployment.

      • Thread Group configuration (users, ramp-up, loop count)
      • HTTP Samplers and logic controllers
      • Performance metrics (Response Time, Throughput, Errors %)
      • Load Test vs Stress Test vs Spike Test
      • Viewing and analyzing results with “View Results Tree”
      ▶ Watch JMeter Tutorial

      Cypress

      Cypress is a front-end end-to-end (E2E) testing tool that provides real-time test runner, instant feedback, and automatic waits without the need for manual timeouts.

      • Real-time test execution with instant feedback
      • Automatic waits (no need for manual timeouts)
      • DOM interaction and UI validation
      • Powerful selector engine
      • Easy debugging with time-travel snapshots
      ▶ Watch Cypress Tutorial

      Jira

      Jira is an agile project management tool used for issue tracking, sprint planning, and team workflow management throughout the development lifecycle.

      • Agile project management basics
      • Creating tasks, bugs, and user stories
      • Sprint planning and backlog management
      • Tracking workflow and issue lifecycle
      • Team collaboration and progress monitoring
      ▶ Watch Jira Tutorial

      About the CTFL Certification

      This section provides a detailed explanation of the six CTFL chapters covered during training.

      Scroll down to get your quiz
      ↓

      Chapter 1 – Fundamentals of Testing

      This chapter introduces the importance of software testing and explains how it helps identify defects early, improve quality, and ensure products meet user expectations.

      It differentiates between testing (finding defects) and debugging (fixing them), while highlighting the roles of verification and validation.

      It also discusses the Seven Testing Principles — such as early testing, defect clustering, and the pesticide paradox — which guide testers toward effective quality assurance practices.

      Chapter 2 – Testing Throughout the SDLC

      This chapter shows how testing integrates into every phase of the Software Development Life Cycle (SDLC), from requirements to maintenance, promoting the Shift-Left mindset.

      It explains key development models — Waterfall, V-Model, Agile, and DevOps — and how testing adapts to each.

      You’ll also learn about the four test levels: Component, Integration, System, and Acceptance, as well as test types like functional, non-functional, structural, and regression testing.

      Chapter 3 – Test Process

      This chapter focuses on the structured testing workflow that ensures consistency and traceability.

      It includes six main activities: Planning, Monitoring, Analysis, Design, Implementation, Execution, and Completion. Each step helps define what to test, how to test it, and how to measure success.

      The use of Entry and Exit Criteria ensures that testing starts and ends at the right time, based on clear quality goals.

      Chapter 4 – Test Techniques

      This chapter teaches how to design effective test cases using Black-box, White-box, and Experience-based techniques.

      Black-box methods include Equivalence Partitioning, Boundary Value Analysis, Decision Tables, State Transition, and Use Cases. These focus on validating behavior without knowing internal code.

      White-box techniques like Statement and Decision coverage focus on code structure, while experience-based techniques rely on tester intuition (Error Guessing, Exploratory Testing).

      Chapter 5 – Test Planning & Estimation

      This chapter emphasizes the importance of strategic test planning — defining objectives, scope, risks, and resources for an efficient and controlled testing effort.

      It introduces estimation techniques like Ratio-based, Extrapolation, Wideband Delphi, and Three-point estimation, helping teams allocate realistic time and effort.

      A well-structured test plan leads to predictable schedules, cost efficiency, and higher test coverage.

      Chapter 6 – Test Tools

      The final chapter introduces tools that support various testing activities — from planning to automation and reporting.

      Categories include Test Management (TestRail, Zephyr), Defect Tracking (Jira, Bugzilla), Automation (Selenium, Playwright), and Performance Testing (JMeter, k6).

      Tools increase efficiency and accuracy but require proper selection, setup, and maintenance to deliver full value.
